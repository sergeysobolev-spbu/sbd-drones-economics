# Назначение

Брокер сообщений (БС) будет использоваться для передачи сообщений между компонентами.

Посколько это учебный пример и инфраструктура, допускается использование брокера сообщений для взаимодействия как между внешними, так и внутренними сервисами.

## Используемые БС

В проекте могут использоваться различные БС. Например, Kafka и Mosquitto. 

В связи с этим необходимо реализовать уровень абстракции, который позволит на уровне конфигурации выбирать конкретный используемый БС и его параметры.

Причины для такой гибкости 

- Kafka лучше подходит для масштабирования и цифрового двойника, позволяет выполнять анализ и повтор всех сообщений, т.к. сообщения не будут удаляться.
- Mosquitto в свою очередь лучше приспособлен к работе с реальным оборудованием (учебными дронами)
